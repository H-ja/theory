Question 5) دستور git-status چه اطلاعاتی نمایش می دهد؟
دستور git status برای برسی وضعیت و تغییرات انجام شده استفاده میشود که شامل :
1.	1. شاخه (branch) ای که در آن هستید.
2.	2. وضعیت فایل های stage نشده (unstaged changes) .
3.	3. وضعیت فایل های stage شده و آماده برای کامیت (staged changes) .
4.	4. فایل های جدید که گیت آن ها را دنبال نمی کند (untracked files) .

Question 7) دستور git-fetch چه فرقی با git-pull دارد؟
تفاوت ناچیزی بین دستور git fetch و git pull وجود دارد.
دستور git fetch تغییرات را از ریموت دریافت می کند ولی کد محلی را تغییر نمی دهد.
دستور git pull تغییرات را از ریموت دریافت می کند و سپس بصورت خودکار تغییرات را با کد محلی ادغام (merge) می کند.

Question 8) تفاوت دستور commit و commit amend چیست؟
این دو دستور از نظر نگارشی مشابه ولی از دیدگاه کاربرد متفاوت هستند.
دستور commit برای ثبت کردن تغییرات ثبت شده (staged changes)  استفاده میشود ولی دستور commit amend برای ویرایش آخرین کامیت استفاده می شود.
یکی از کاربرد های commit amend ویرایش نام آخرین کامیت با استفاده از دستور git commit  --amend -m 'X' است یا اگر فراموش کردید برخی فایل ها را به commit خود اضافه کنید می توان با دستور git commit  - -amend - -no-edit این کار را انجام داد.
 no-edit : اخرین تغییرات که اضافه شده اند را به کامیت قبلی اضافه می کند ولی پیغام کامیت را تغییر نمی دهد.
اگر بخواهیم پیغام کامیت را تغییر دهیم باید از دستور git commit –amend -m 'new X' استفاده کنیم.

Question 9) چرا نوشتن یک commit message خوب مهم است؟ توضیح دهید.
یک commit message خوب باعث میشه افراد بهتر بفهمند در هر کامیت چه تغییراتی ایجاد شده و مهم تر از همه "چرا یک تغییر ایجاد شده است؟" هم چنین این امر منجر به رفع بهتر باگ ها و مشکلاتی میشه که در برنامه وجود دارد چون حدس اینکه کدام بخش ممکن است باعث ایجاد این مشکلات شود راحت تر خواهد بود.
